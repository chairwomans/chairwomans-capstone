# ☁️ Team Ground Rule

## 1. 연락

- 모든 공식적인 소통은 **디스코드**를 통해 진행하므로, **알림**은 항상 켜두고 수시로 확인한다.
- 회신이 필요한 경우에는 새벽 시간대를 제외하고 **3시간 이내**에 답장한다.
    - **부재 예정**일 경우 사전에 공지한다.
    - 장시간 응답이 어려운 경우 **상태 메시지**로 알린다.
- 공지 및 안내 사항은 반드시 **정독**한 후 **확인 표시**(✅)를 남긴다.
- 중요한 사안은 반드시 **멘션**(@)을 사용한다.

## 2. 회의

- 정기 회의는 **매주 월요일 18:30**에 진행한다.
    - 회의 **지각** 시 사전에 공유한다.
    - **불참** 시 사전에 공유하고 **회의록**을 반드시 확인한다.
- 회의는 **대면**으로 진행하되, 부득이한 경우 **디스코드/회의방**에서 진행한다.
- 번갈아가면서 **회의록 담당자**를 지정하고, 모든 회의록은 **노션**에 지정된 **양식**에 따라 기록한다.

## 3. 프로젝트 진행

- 원활한 진행을 위해 중요한 **개인 장기 일정**은 사전에 공유한다.
- 모든 업무는 팀에서 정한 **마감 기한** 내에 완료하는 것을 원칙으로 한다.
    - 일정 **지연**이 예상될 경우 즉시 공유한다.
- 담당자가 지정된 업무는 해당 **담당자**가 책임진다.
- 이해되지 않는 업무는 임의로 진행하지 않고 반드시 **확인 후 진행**한다.

## 4. 스터디

- 스터디 내용은 개별로 **주 1회** 공유하며, 공유 간격은 1주일을 넘기지 않고 노션에 진도와 코드 스크린샷을 업로드한다.
- 스터디 중 이해되지 않는 부분은 **즉시 질문**한다.

## 5. 코드 관리

- 모든 코드는 지정된 **Git 저장소**에서 관리한다.
- **main 브랜치**에 직접 푸시하지 않는다.
- 모든 작업은 **브랜치 생성** 후 **Pull Request**를 통해 병합한다.
    - Pull Request는 최소 **1인 이상 리뷰 및 승인** 후 병합한다.
- **커밋 메시지**는 작업 내용을 명확히 작성하며, 정해진 **컨벤션**을 따른다.
- **실험용 코드**와 **최종 코드** 버전을 명확히 구분한다.

## 6. 실험 관리

- 모든 **실험 과정**은 **노션**에 정리한다.
- **실패한 실험**도 기록하고 공유한다.
